# DoublePlus 设计说明

## 目标与范围

DoublePlus 是一款 macOS 菜单栏应用，通过 **Finder Sync 扩展** 在 Finder 的右键菜单中提供类似 Windows「新建」的体验：在文件夹空白处或选中项上下文中创建文本文件、Markdown 文件或文件夹。主程序仅常驻菜单栏（无 Dock 图标），用于提示用户启用扩展、打开扩展管理界面以及退出应用。

## 模块划分

| 模块 | 路径 | 职责 |
|------|------|------|
| 主应用 | `DoublePlus/` | `MenuBarExtra` UI、`LSUIElement`、关于/退出、调用 `FIFinderSyncController.showExtensionManagementInterface()` |
| 共享逻辑 | `DoublePlusShared/` | 动作协议、注册表、文件创建与命名、App Group 标识常量 |
| Finder 扩展 | `DoublePlusFinderSync/` | 子类化 `FIFinderSync`，实现 `menu(for:)`，组装「DoublePlus」子菜单并执行已注册动作 |

## Finder Sync 与主程序职责

- **Finder Sync 扩展**在 Finder 进程侧运行，是唯一能在 Finder 右键菜单中注入条目的标准方式。扩展通过 `FIFinderSyncController` 获取 `targetedURL()`、`selectedItemURLs()`，并在用户选择菜单项时执行创建操作。
- **主程序**不直接参与 Finder 菜单，负责无 Dock 的菜单栏体验，并引导用户在系统设置中启用扩展。

## 动作协议与注册表

- **`FinderContextAction`**：定义 `id`、`title`、`supports(menuKind:)`、`validate(context:)`、`perform(context:)`。不依赖 `FinderSync` 框架中的 `FIMenuKind`，而是用 **`FinderMenuKind`** 抽象，便于主程序与扩展共用源码且避免主 Target 链接 FinderSync（主程序仅因 UI 需要链入 FinderSync 以调用扩展管理 API）。
- **`FinderActionContext`**：携带菜单种类、目标 URL、选中项 URL 列表。
- **`ActionRegistry`**：单例，启动时注册内置动作。后续若要增加「右键分享」等功能，可新增实现 `FinderContextAction` 的结构体/类型，并在 `registerBuiltIn()` 或单独初始化路径中 `register(...)`，无需改动菜单组装逻辑（`FinderSync.swift` 仅遍历注册表）。

## App Group

- 标识符：`group.Ncc.DoublePlus`（见 `SharedAppGroup.identifier` 与双方 entitlements）。
- 当前版本尚未把模板列表持久化到共享容器，预留用于日后在主应用编辑模板、扩展读取同一配置。

## 监视目录与菜单上下文

- 扩展在 `init` 中将 `directoryURLs` 设为根卷 `/` 的 URL，以便在常见 Finder 场景下收到回调（与 Apple 示例一致；若需缩小范围可改为用户主目录等集合）。
- **`lastMenuKind`** 在 `menu(for:)` 中更新，在执行 `performMenuAction` 时与控制器提供的 URL 一起构成完整上下文。

## 错误处理与命名策略

- 创建失败时扩展使用 `NSAlert` 展示 `LocalizedError` 描述。
- **`FileCreationService`** 在重名时自动追加序号（如 `未命名 2.txt`、`新建文件夹 2`）。

## 后续扩展点

- **分享**：新增 `FinderContextAction` 实现，在 `perform` 中调用 `NSSharingService`，并对 `FinderMenuKind` 为选中项的场景做 `validate`。
- **可配置模板**：主应用写入 App Group 容器（如 JSON），扩展启动时读取并动态注册动作或仅影响默认扩展名与文件名前缀。

## 构建与签名

- 工程含两个 Target：应用与 `DoublePlusFinderSync` 扩展；应用通过 **Embed Foundation Extensions** 将 `DoublePlusFinderSync.appex` 嵌入 `Contents/PlugIns`。
- 本地无描述文件时需由 Xcode 自动签名；命令行可加 `-allowProvisioningUpdates` 或仅在 Xcode 内构建。
