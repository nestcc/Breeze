# Breeze 使用说明

## 安装与运行

1. 使用 Xcode 打开 `Breeze.xcodeproj`，在顶部 scheme 菜单中选择 **Breeze**（主应用），连接你的 Apple 开发团队后运行（⌘R）。
2. 若出现 **Choose an app to run?**，多半是选中了 **BreezeFinderSync** 扩展 scheme：扩展不能单独运行，必须选 **Breeze** 作为主程序；或把 **Product → Destination** 设为 **My Mac**。
3. 首次运行后，菜单栏会出现 **Breeze** 图标（当前为系统 SF Symbol「文档加号」样式）。

## 启用 Finder 扩展

Breeze 使用的是 **Finder Sync** 扩展（系统扩展点 `com.apple.FinderSync`），**不是**「文件提供程序 / File Provider」类扩展。系统设置里的 **「文件提供程序」** 列表主要面向网盘、同步类 **File Provider**，**里面没有 Breeze 是正常现象**，请不要只在那一栏里找。

推荐顺序如下。

### 1. 菜单栏里的「管理 Finder 扩展…」（优先）

从菜单栏打开 Breeze 菜单，点击 **「管理 Finder 扩展…」**。系统会打开与 Finder 相关的扩展管理界面（具体布局随 macOS 版本变化）。在列表中找到 **Breeze** / **Breeze Finder** 并打开开关。

### 2. 系统设置里搜索与「登录项与扩展」

- 在 **系统设置** 顶部搜索 **扩展**、**Finder** 或 **Breeze**。
- 打开 **通用 → 登录项与扩展**，查看 **Finder** 等分组下是否出现 Breeze（不同系统版本文案不一致，**不要**与「文件提供程序」混淆）。

### 3. macOS Ventura / Sonoma（若仍有旧版界面）

**系统设置 → 隐私与安全性 → 扩展 → 添加的扩展 → Finder**，勾选 Breeze。在 **macOS Sequoia** 上，部分机器侧边栏里**没有**「隐私与安全性 → 扩展」，属界面调整。

### 4. 终端启用（开发/排查用，Apple 开发者论坛中讨论过的做法）

若图形界面里始终找不到开关，可先**至少运行过一次 Breeze**，再在「终端」中执行：

```bash
pluginkit -m -v -p com.apple.FinderSync
```

确认列表中有 `Ncc.Breeze.FinderSync`（路径应指向你的 `Breeze.app` 内的 `BreezeFinderSync.appex`）。再执行：

```bash
pluginkit -e use -p com.apple.FinderSync -i Ncc.Breeze.FinderSync
```

然后可再运行第一条命令，行首应为 `+` 表示已启用。若仍无菜单，可尝试**重新启动 Finder**。

若未出现，可先退出并重新运行主应用，或注销后重试。Sequoia **15.0–15.1** 曾短暂缺少图形界面，需升级到 **15.2+**。

## 在 Finder 中使用「新建」

1. 打开任意 Finder 窗口，进入目标文件夹。
2. 在窗口**空白处**右键（或 Control-点按），在菜单中找到 **Breeze** 子菜单。
3. 可选操作包括：
   - **新建文本文件**（`.txt`）
   - **新建 Markdown 文件**（`.md`）
   - **新建文件夹**
4. 若已选中文件或文件夹再右键，部分操作会以**当前选中项所在目录**为创建位置（与 Windows「在当前目录新建」类似）。

若某菜单项未出现，多为当前上下文不满足条件（例如无法解析目标文件夹）。

## 菜单栏操作

- 下拉菜单中有简短说明、扩展未启用时的提示、**管理 Finder 扩展** 以及 **退出 Breeze**（⌘Q）。

## 如何更换菜单栏图标

当前使用 SwiftUI `MenuBarExtra` 的 **`systemImage:`** 参数，占位为 `doc.badge.plus`。

**改为自定义图片：**

1. 在 Xcode 中打开 **Assets.xcassets**，新建 **Image Set**，命名为例如 `MenuBarIcon`。
2. 将 **Render As** 设为 **Template Image**，拖入 1x / 2x 图稿（菜单栏约 18pt，建议提供 @2x）。
3. 在 `BreezeApp.swift` 中，将  
   `MenuBarExtra("Breeze", systemImage: "doc.badge.plus")`  
   改为使用资源名，例如：  
   `MenuBarExtra(content: { MenuBarContentView() }) { Image("MenuBarIcon") }`  
   或使用 `Label("Breeze", image: "MenuBarIcon")` 等等价写法（以你采用的 `MenuBarExtra` 初始化器为准）。

**说明：** 菜单栏图标与 **AppIcon**（应用图标）是两套资源；已启用 **LSUIElement** 时 Dock 一般不显示应用图标，但「关于本机」等仍可能使用 AppIcon。

## 常见问题

- **「文件提供程序」里没有 Breeze：** **预期行为**。Breeze 是 **Finder Sync**，不是 File Provider；请用 **管理 Finder 扩展…**、系统设置搜索，或上文 **终端 pluginkit** 方式。
- **隐私与安全性里没有「扩展」：** 在 **macOS Sequoia** 上很常见；请用 **通用 → 登录项与扩展**、**管理 Finder 扩展…** 或 **pluginkit**。
- **扩展已勾选但仍无菜单：** 尝试重启 Finder（可按住 Option 右键 Finder 图标选「重新开启」），或重新登录系统。
- **无法创建文件：** 确认对目标文件夹有写权限；部分受保护系统目录可能无法写入。
- **沙盒与签名：** 应用与扩展均启用 Sandbox 并配置同一 App Group；分发时需正确签名与公证（若上架 Mac App Store，需遵守 Apple 对扩展的审核要求）。
